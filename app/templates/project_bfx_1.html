BFX 專案及環境

Overview

需求說明

  • 用理賠X光骨折(Bone Fracture X-ray)POC案的影像，內部作一版模型。
  • 嘗試的成果與困難皆可回饋理賠，協助判斷POC案成效。
  • 同時提升部門對影像辨識、環境建置與驗測流程的能力。

    • 資料敘述

    資料主要有 Dicom 及 Array 兩類：



    環境說明

    1. work資料夾內的東西是共用的
    2. 程式碼建議放進9h資料夾內，避免遺失
    3. 資料建議不要瘋狂複製，空間有限
    4. 需要的套件直接在terminal pip install即可
    5. 每個jupyter有專用的GPU，請科內自行協調資源使用
    6. 如果特殊需求一定要自己一張GPU，請洽工程組
    7. Jupyter lab 有非常多插件，可以在左側的 extension manager 自行加入
    8. 未盡事項請參考 jupyter 開發環境 work 資料夾內文件

    驗測標準

    請預測 test data 的部位結果，分別是 0、1、2、3 四種結果，代表無、龜裂、不完全骨折、完全骨折。分數計算方式使用 accuracy 為指標。上傳檔案範本請參考附件 submission.csv。


    可以參考附件的 submission.csv，第一個欄位是BFX編號，第二個欄位是結果，只會有 0、1、2、3 四種結果，分別代表無、龜裂、不完全骨折、完全骨折。

    附件區

    本次 dataset 極為龐大，建議使用下面提供的 jupyter 環境進行開發，若要下載請慎思。

    dicom.zip 14.3GB dicom資料
    array.zip 14.2GB array資料

    BFX_範例.ipynb 163.4KB 程式範例，記得絕對路徑要改一下
    submission.csv 8.5KB 上傳csv範例
