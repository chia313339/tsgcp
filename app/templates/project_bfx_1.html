BFX 專案及環境

Overview

需求說明

  • 用理賠X光骨折(Bone Fracture X-ray)POC案的影像，內部作一版模型。
  • 嘗試的成果與困難皆可回饋理賠，協助判斷POC案成效。
  • 同時提升部門對影像辨識、環境建置與驗測流程的能力。

    • 資料敘述

    資料主要有 Dicom 及 Array 兩類：



    環境說明

    1. work資料夾內的東西是共用的
    2. 程式碼建議放進9h資料夾內，避免遺失
    3. 資料建議不要瘋狂複製，空間有限
    4. 需要的套件直接在terminal pip install即可
    5. 每個jupyter有專用的GPU，請科內自行協調資源使用
    6. 如果特殊需求一定要自己一張GPU，請洽工程組
    7. Jupyter lab 有非常多插件，可以在左側的 extension manager 自行加入
    8. 未盡事項請參考 jupyter 開發環境 work 資料夾內文件

    驗測標準

    1. BFX_codedkeys.csv 是BFX資料對照表
    2. 訓練過程中需要參照的答案有4個欄位
      • CODED_PART: 部位 (24個部位(包含無)，plz refer to範例)
      • CODED_LEVEL: 嚴重程度 (0/1/2/3)
      • CODED_APLY: 受編，上傳答案的ＫＥＹ
      • filename: 影像檔案名
    3. RAW DICOM的資料已經先被整理過，整理成DICOM或是純ARRAY格式。
    4. ARRAY的訓練放在 /array/train/... ，檔案格式為joblib 讀取請見範例。
    5. 上傳答案需包含CODED_APLY, CODED_PART, CODED_LEVEL。
    6. 幾個欄位註記：
      • IS_NEG : 是不是白底照
      • APLY_IS_UP_EXM : 受編是否是屬於上肢
      • APLY_IS_LOW_EXM :受編是否是屬於下肢

    如有未盡事項請參考 notebook 環境內的文件。

    上傳附件格式請參考附件的 BFX_EXAMPLE.csv，內容請詳閱驗測標準，排行榜預設以 acc_part_overall 作為指標。
